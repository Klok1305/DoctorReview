# Пульс клиники

Локальное Windows-приложение для загрузки выгрузок 1С, расчёта показателей врачей и отделений и пакетного формирования отчётов.

## Скачать приложение

### Файл 1 — portable-версия: запустить сразу

[⬇️ Скачать DoctorReview Portable 1.18.2](https://github.com/Klok1305/DoctorReview/releases/download/v1.18.2/DoctorReview-Portable-1.18.2-x64.exe)

После скачивания дважды щёлкните по `.exe` — приложение запустится без установки.
Этот вариант нужен для быстрого запуска, проверки программы или использования с
флешки. Ярлыки и деинсталлятор не создаются, потому что portable-версия ничего не
устанавливает. Для удаления достаточно закрыть программу и удалить её `.exe`.

### Файл 2 — установщик: для постоянной работы

[⬇️ Скачать установщик DoctorReview 1.18.2](https://github.com/Klok1305/DoctorReview/releases/download/v1.18.2/DoctorReview-Setup-1.18.2-x64.exe)

Это рекомендуемый вариант. Установщик сам устанавливает программу и все необходимые
компоненты — отдельно устанавливать Node.js или другие программы не нужно. Он создаёт
ярлыки на рабочем столе и в меню «Пуск», подключает автоматические обновления и
добавляет стандартный деинсталлятор Windows.

Удалить установленную программу можно через **Параметры Windows → Приложения →
Установленные приложения → Пульс клиники → Удалить**. Деинсталлятор удаляет файлы
программы, но сохраняет выбранную рабочую папку с базой, отчётами и резервными копиями.

## Что получает пользователь

- ярлык «Пульс клиники» на рабочем столе;
- выбор рабочей папки при первом запуске;
- отдельные настраиваемые папки для входящих файлов, результатов и резервных копий;
- база SQLite, которая сохраняется автоматически и рассчитана на многомесячную работу;
- защита от повторной загрузки одного и того же файла по SHA-256;
- резервная копия перед каждым импортом, восстановлением и обновлением;
- отдельный PDF для каждого врача и каждого отделения за выбранный месяц;
- все текущие PDF одного месяца в одной папке, предыдущие версии — в архиве;
- сводная Excel-таблица месяца;
- отдельные установщик Windows и portable-версия.

## Установка

### Обычная установка — рекомендуется

1. Откройте страницу [последнего релиза](https://github.com/Klok1305/DoctorReview/releases/latest).
2. В разделе **Assets** скачайте файл `DoctorReview-Setup-<версия>-x64.exe`.
   Файлы `latest.yml` и `.blockmap` скачивать не нужно — они используются системой
   автоматических обновлений.
3. Запустите скачанный установщик. Если Windows SmartScreen покажет предупреждение,
   нажмите «Подробнее» → «Выполнить в любом случае».
4. Выберите папку установки и завершите установку. На рабочем столе появится ярлык
   «Пульс клиники».
5. При первом запуске выберите отдельную локальную рабочую папку для базы и отчётов.
   Не используйте OneDrive. Если особых требований нет, оставьте предложенную папку
   `C:\Users\<имя пользователя>\Пульс клиники`.

Если уже установлена предыдущая версия, скачайте актуальный установщик и в старой
версии нажмите «Установить обновление из файла». После перехода на 1.18 следующие
обновления смогут загружаться автоматически. Рабочая
база и созданные отчёты при переустановке не удаляются.

### Portable-версия

Скачайте `DoctorReview-Portable-<версия>-x64.exe`, поместите файл в отдельную папку
и запустите без установки. Рабочие данные всё равно храните в выбранной папке,
отдельно от portable-файла. Для обновления portable-версии замените старый `.exe`
новым файлом из последнего релиза.

## Папки и хранение

По умолчанию рабочая папка — `C:\Users\<имя пользователя>\Пульс клиники`:

```text
Пульс клиники
├── Входящие
├── База
│   └── оценка-врачей.sqlite
├── Результаты
│   └── 2026-01
├── Резервные копии
└── Журналы
```

В `config.json` внутри `%APPDATA%\Пульс клиники` хранятся только выбранные пути и настройки приложения. После обновления со старой версии приложение автоматически использует прежнюю папку настроек. Основные данные находятся в рабочей SQLite-базе. Выгрузки 1С и данные пациентов не отправляются в интернет.

## Сборка для разработки

Требуются Node.js 22+ и pnpm 11+.

```powershell
pnpm install --frozen-lockfile
pnpm test
pnpm run test:smoke
pnpm run test:pdf
pnpm run dist:all
```

Или одной командой:

```powershell
.\scripts\build-desktop.ps1 -SmokeTest
```

Готовые файлы появляются в `dist`. Инструкция для пользователя находится в [docs/ИНСТРУКЦИЯ.md](docs/ИНСТРУКЦИЯ.md), порядок выпуска обновлений — в [docs/ОБНОВЛЕНИЕ.md](docs/ОБНОВЛЕНИЕ.md).

## Безопасность репозитория

`.gitignore` работает как белый список: в Git разрешены только код, тесты,
сборочные сценарии, иконки и документация. Таблицы, архивы, SQLite-базы, PDF,
журналы, локальные настройки, готовые установщики и сертификаты не добавляются.

Команда `pnpm run audit:repo` повторно проверяет все публикуемые файлы, их тип,
размер и отсутствие типовых секретов. Та же проверка обязательна в GitHub Actions.

## Облачные обновления

При отправке тега `v<версия>` GitHub Actions собирает Windows-установщик и portable
пакет и публикует их в [GitHub Releases](https://github.com/Klok1305/DoctorReview/releases).
Приложение читает оттуда `latest.yml`, загружает новую версию и перед установкой
создаёт резервную копию рабочей базы. Подробный порядок выпуска находится в
[docs/ОБНОВЛЕНИЕ.md](docs/ОБНОВЛЕНИЕ.md).
