# Оценка врачей

Локальное Windows-приложение для загрузки выгрузок 1С, расчёта показателей врачей и отделений и пакетного формирования отчётов.

## Что получает пользователь

- ярлык «Оценка врачей» на рабочем столе;
- выбор рабочей папки при первом запуске;
- отдельные настраиваемые папки для входящих файлов, результатов и резервных копий;
- база SQLite, которая сохраняется автоматически и рассчитана на многомесячную работу;
- защита от повторной загрузки одного и того же файла по SHA-256;
- резервная копия перед каждым импортом, восстановлением и обновлением;
- отдельный PDF для каждого врача и каждого отделения за выбранный месяц;
- все текущие PDF одного месяца в одной папке, предыдущие версии — в архиве;
- сводная Excel-таблица месяца;
- установщик Windows и portable-версия в одном `.exe`.

## Папки и хранение

По умолчанию рабочая папка — `C:\Users\<имя пользователя>\Оценка врачей`:

```text
Оценка врачей
├── Входящие
├── База
│   └── оценка-врачей.sqlite
├── Результаты
│   └── 2026-01
├── Резервные копии
└── Журналы
```

В `config.json` внутри `%APPDATA%\Оценка врачей` хранятся только выбранные пути и настройки приложения. Основные данные находятся в рабочей SQLite-базе. Выгрузки 1С и данные пациентов не отправляются в интернет.

## Сборка для разработки

Требуются Node.js 22+ и pnpm 11+.

```powershell
pnpm install --frozen-lockfile
pnpm test
pnpm run test:smoke
pnpm run test:pdf
pnpm run dist:all
```

Или одной командой:

```powershell
.\scripts\build-desktop.ps1 -SmokeTest
```

Готовые файлы появляются в `dist`. Инструкция для пользователя находится в [docs/ИНСТРУКЦИЯ.md](docs/ИНСТРУКЦИЯ.md), порядок выпуска обновлений — в [docs/ОБНОВЛЕНИЕ.md](docs/ОБНОВЛЕНИЕ.md).

## Безопасность репозитория

`.gitignore` работает как белый список: в Git разрешены только код, тесты,
сборочные сценарии, иконки и документация. Таблицы, архивы, SQLite-базы, PDF,
журналы, локальные настройки, готовые установщики и сертификаты не добавляются.

Команда `pnpm run audit:repo` повторно проверяет все публикуемые файлы, их тип,
размер и отсутствие типовых секретов. Та же проверка обязательна в GitHub Actions.

## Облачные обновления

При отправке тега `v<версия>` GitHub Actions собирает Windows-установщик и portable
пакет и публикует их в [GitHub Releases](https://github.com/Klok1305/DoctorReview/releases).
Приложение читает оттуда `latest.yml`, загружает новую версию и перед установкой
создаёт резервную копию рабочей базы. Подробный порядок выпуска находится в
[docs/ОБНОВЛЕНИЕ.md](docs/ОБНОВЛЕНИЕ.md).
